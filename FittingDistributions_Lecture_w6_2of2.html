Introduction¶

This far, most data we have worked with are normally distributed. But in most cases data, especially in certain disciplines, will not follow a normal distribution. In this lecture we wish to identify the tools that will allow us to identify a distribution that best describes the data we have at hand.

While we will not be able to perform t-tests as we know them if the data are not normally distributed (how to transform data will not be dealt here), identifying a distribution that best describes a dataset will allow us to simulate scenarios and create projections which we can use to support decision making.

  • I will generate a sequence of 100000 numbers distributed following a Weibull distribution. Note: The Weibull distribution has more parameters than the normal distribution (mean and variance). Those are namely, (i) Location (you can think of this as the mean), (ii) scale (you may think of this as the variance), (iii) shape (with some caution you may think of this as the skewness)

    http://www.mathbitsnotebook.com/Algebra1/StatisticsData/STShapes.html

We will use the following parameters to create this distribution:
c = location = 0 (shift from origin),
b = scale = 2 and
a = shape = 1

Some maths to have in mind:

The Weibull distribution with shape parameter a and scale parameter b has density given by

$f(x) = (a/b) (x/b)^(a-1) exp(- (x/b)^a) for x > 0$

The cumulative distribution function is (remember capital F for the cummulative distribution)

$F(x) = 1 - exp(- (x/b)^a) on x > 0$

Theoretical moments for Weibull distributions are:

\begin{eqnarray} Mean(X) &=& E(X) = c+b (1 + 1/a)\\ Var(X) &=& b^2 * ((1 + 2/a) - ((1 + 1/a))^2) \end{eqnarray}

The fitting criterion is always the LogLikelihood. The model with the highest LogLikelihood is the best fit model always. What we need to be cautious of nonetheless is the number of parameters used to estimate a model. If we use a model with many number of parameters it is likely that the model will have a better fit but it is also likely that we overfit. This means that the model may be good for explainign the history but not for explaining the out-of-sample data. To this end we need to consider the so called Akaike Information Criterion (AIC) and Bayesian Information Criterion (BIC). These penalise models based on the number of parameters (and the likelihood value). The model with the lowest is usually a better candidate.

So to summarise we need: (i) Higher Likelihood, and (ii) Lower BIC/AIC.

The highest liklihood and lower BIC/AIC here is the Weibull (almost identical to the gamma to be honest - this is because of the nature of the two distributions that are special cases of each other).

These information are enough for one to support the Weibull distribution as a good choise - when compared to these three. Let's prooce this visually too. Futher, progress in your studies you will see more criterion that can assist you in selecting a distribution fit.
